# Contributing to OpenCode-OpenHands Integration

We love your input! We want to make contributing to this project as easy and transparent as possible.

## Development Process

1. Fork the repo
2. Create your feature branch (`git checkout -b feature/AmazingFeature`)
3. Make your changes
4. Test thoroughly
5. Commit your changes (`git commit -m 'Add AmazingFeature'`)
6. Push to the branch (`git push origin feature/AmazingFeature`)
7. Open a Pull Request

## Pull Request Process

1. Update the README.md with details of changes if needed
2. Update the documentation in `docs/` if you changed architecture
3. The PR will be merged once you have approval from maintainers

## Code Style

- **Python:** Follow PEP 8, use Black formatter
- **JSON:** 2-space indentation
- **Documentation:** Markdown with clear headings

## Testing

Before submitting a PR:

```bash
# Test MCP server
python mcp_server.py
# In another terminal:
# Test @openhands health_check in OpenCode
```

## Reporting Bugs

Use GitHub Issues and include:

1. **Environment:** Python version, OS, OpenCode version
2. **Steps to reproduce**
3. **Expected behavior**
4. **Actual behavior**
5. **Logs** (if applicable)

## Feature Requests

We love feature ideas! Open an issue with:

1. **Use case:** What problem does it solve?
2. **Proposed solution:** How should it work?
3. **Alternatives:** What other approaches did you consider?
